# Contributing to JAI Lab projects

Thank you for helping build open infrastructure for molecular discovery.

## Good contributions

Useful contributions include:

- fixing documentation
- adding examples
- improving reproducibility
- adding tests
- cleaning datasets
- proposing benchmark tasks
- reporting issues in workflows
- contributing notebooks or analysis scripts
- adding citations and references

## Scientific contribution standard

When contributing scientific content, include:

1. the scientific question
2. data source and license
3. preprocessing steps
4. assumptions and limitations
5. expected output
6. how to reproduce the result
7. suggested citation

## Pull request checklist

- [ ] The contribution has a clear purpose.
- [ ] Data sources are documented.
- [ ] No private or restricted data is included.
- [ ] Examples run or are clearly marked as drafts.
- [ ] Dependencies are documented.
- [ ] Scientific limitations are stated.
- [ ] README or docs are updated.

## Safety

Do not contribute content that enables illegal drug synthesis, evasion of law enforcement, weaponization, or unsafe chemical misuse. Analytical, educational, and harm-reduction-oriented sensing tools should remain focused on detection, characterization, and public-interest scientific understanding.
