# Open science playbook

## Mission

Make scientific work reusable by default.

## Dataset checklist

- [ ] Data source documented
- [ ] License documented
- [ ] Schema documented
- [ ] Units included
- [ ] Experimental conditions included
- [ ] Controls included when relevant
- [ ] Missing values documented
- [ ] Uncertainty documented when available
- [ ] Preprocessing script included
- [ ] Example notebook included

## Benchmark checklist

- [ ] Task definition
- [ ] Input format
- [ ] Output format
- [ ] Train/validation/test split
- [ ] Evaluation metric
- [ ] Baseline model
- [ ] Known failure modes
- [ ] Citation

## Model checklist

- [ ] Training data
- [ ] Feature representation
- [ ] Architecture or method
- [ ] Hyperparameters
- [ ] Metrics
- [ ] Uncertainty estimate if relevant
- [ ] Intended use
- [ ] Limitations

## Documentation principle

A collaborator should be able to understand the scientific purpose before reading code.
