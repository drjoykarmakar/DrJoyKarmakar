# Profile strategy

## Goal

Turn the GitHub profile into a serious public research landing page that communicates:

1. who Joy Karmakar is
2. what JAI Lab is building
3. what visitors can use today
4. what the research thesis is
5. how collaborators can engage

## Core positioning

Joy Karmakar is a medicinal chemist building open infrastructure for AI-powered molecular discovery.

JAI Lab is not only a portfolio. It is a public research ecosystem built around datasets, benchmarks, software, and molecular AI.

## Audience

- faculty search committees
- biotech and AI drug discovery teams
- molecular AI researchers
- analytical chemists
- fluorescent probe scientists
- open-source contributors
- funders and translational collaborators

## Rules

- Lead with clarity, not decoration.
- Use concrete artifacts: repositories, datasets, benchmarks, software, papers.
- Avoid unsupported claims like "world first" unless documented.
- Put the strongest public evidence on the first screen.
- Use consistent names, colors, and section order.

## Best profile repo setup

The root `README.md` is the profile. All supporting assets should sit in `assets/`, `docs/`, `templates/`, and `repository-readmes/`.

Avoid uploading a package README as the root README. The root README should always be the public landing page.
