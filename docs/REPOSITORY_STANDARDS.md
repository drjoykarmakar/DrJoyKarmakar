# Repository standards

Every JAI Lab repository should feel like part of the same scientific ecosystem.

## README structure

1. Hero or short identity line
2. Problem statement
3. What this repository contains
4. Quickstart
5. Repository structure
6. Data or model card
7. Examples
8. Reproducibility
9. Limitations
10. Roadmap
11. Citation
12. Contributing
13. License

## Minimum files

```text
README.md
LICENSE
CITATION.cff
CONTRIBUTING.md
SECURITY.md
.github/ISSUE_TEMPLATE/
examples/
docs/
```

## Scientific reproducibility

Include:

- data source
- preprocessing
- feature generation
- model and parameter details
- random seeds
- evaluation metric
- validation split
- limitations
- citation

## Naming

Use clear names. Avoid unexplained acronyms unless they are core brands.

## Badges

Use badges sparingly. Recommended:

- license
- Python version
- GitHub stars
- status
- DOI or Zenodo when available

## Limitations

Every scientific repository should include a limitations section. This increases credibility.
