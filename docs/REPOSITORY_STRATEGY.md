# Repository Strategy

## Recommended organization structure

```text
.github                         Organization profile and community files
sensorgenome                    Open molecular sensing data infrastructure
dyemind                         AI for fluorescent probe discovery
azai                            Xylazine analytics and forensic chemistry workflows
narcoticsense-ai                Spectroscopy and chemometric intelligence
molecular-discovery-suite       QSAR, docking, SAR, and ADMET workflows
bimanedb                        Bimane chemistry database and tools
paper-organizer                 Literature intelligence and research organization
jai-lab-docs                    Documentation site
jai-lab-assets                  Shared branding and diagrams
```

## Repository README structure

1. Project name and one-line purpose
2. Problem solved
3. Quick start
4. Core features
5. Scientific use cases
6. Data/schema/model documentation
7. Examples
8. Reproducibility
9. Citation
10. Contributing
11. License

## Naming rules

- Use lowercase repository names.
- Use hyphens for readability.
- Keep names short.
- Keep JAI Lab as the organization brand; project names can be specialized.
